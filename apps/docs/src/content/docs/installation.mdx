## Install the package

```bash
npm install trud-calendar
# or
pnpm add trud-calendar
# or
yarn add trud-calendar
```

**Peer dependencies:** `react >=18`, `react-dom >=18`

## Tailwind v4 setup

Import the theme variables **before** Tailwind, and add the `@source` directive so Tailwind generates the required utility classes:

```css
@import "trud-calendar/styles.css";
@import "tailwindcss";
@source "../node_modules/trud-calendar/dist";
```

:::tip[Why import order matters]
trud-calendar's variables are defined inside `@layer trc`. Importing the styles **before** Tailwind ensures `@layer trc` is declared first and gets the lowest cascade priority — so your overrides always win. See [Theming > Overriding variables](/theming/#overriding-variables-with-tailwind-v4) for details.
:::

## Without Tailwind

If you're not using Tailwind, just import the pre-built CSS file:

```css
@import "trud-calendar/styles.css";
```

Or in your JavaScript entry:

```ts
import "trud-calendar/styles.css";
```

## shadcn/ui projects

### shadcn v2 (Tailwind v4)

If your project uses shadcn v2, the calendar inherits your theme automatically. The CSS variables fall back to shadcn's `var(--background)`, `var(--foreground)`, etc.

Just add the `@source` directive and import the styles:

```css
@import "trud-calendar/styles.css";
@import "tailwindcss";
@source "../node_modules/trud-calendar/dist";
```

### shadcn v1 (Tailwind v3)

shadcn v1 defines CSS variables as raw HSL numbers (e.g. `--border: 214.3 31.8% 91.4%`), which are not valid CSS colors on their own. You'll need to map them manually with `hsl()` wrappers. See the [Theming > shadcn v1](/theming/#shadcn-v1-tailwind-v3) section for the full snippet.
