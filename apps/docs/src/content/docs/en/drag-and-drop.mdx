---
title: Drag & Drop
description: Enable drag and drop to move events between time slots.
---

trud-calendar uses the native HTML5 Drag and Drop API — zero extra dependencies.

## Enable drag and drop

Set `enableDnD={true}` and provide an `onEventDrop` handler:

```tsx
<Calendar
  events={events}
  enableDnD
  onEventDrop={(event, newStart, newEnd) => {
    // event: the original CalendarEvent (with id, custom fields, etc.)
    // newStart: "2026-03-15T10:00:00"
    // newEnd: "2026-03-15T11:00:00"
    await api.updateEvent(event.id, { start: newStart, end: newEnd });
  }}
/>
```

## How it works

### Month view
- Drag an event pill from one day cell to another
- The **date changes** but the **original time is preserved**
- Event duration is maintained

### Week / Day view
- Drag an event from one time slot to another
- Both **date and time** change
- Snaps to **15-minute increments**
- Event duration is maintained

## The `onEventDrop` callback

```ts
onEventDrop?: (
  event: CalendarEvent,   // The original event object
  newStart: DateTimeString, // New start: "2026-03-15T10:00:00"
  newEnd: DateTimeString,   // New end: "2026-03-15T11:00:00"
) => void;
```

You receive:
- The full original `CalendarEvent` — use `event.id` to update your database
- New start and end times as ISO 8601 strings — ready to send to your API
- Duration is always preserved (calculated automatically)

## Example: Update database on drop

```tsx
function App() {
  const [events, setEvents] = useState(initialEvents);

  const handleDrop = async (event, newStart, newEnd) => {
    // Optimistic update
    setEvents((prev) =>
      prev.map((e) =>
        e.id === event.id ? { ...e, start: newStart, end: newEnd } : e
      )
    );

    // Persist to backend
    await fetch(`/api/events/${event.id}`, {
      method: "PATCH",
      body: JSON.stringify({ start: newStart, end: newEnd }),
    });
  };

  return (
    <Calendar
      events={events}
      enableDnD
      onEventDrop={handleDrop}
    />
  );
}
```
